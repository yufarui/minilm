从零搭建 端到端 LLM 训练管线：自定义 Decoder-only 因果模型（含 MoE、RoPE、Flash Attention 等）、自训 BBPE Tokenizer、配置驱动的预训练与 SFT 数据清洗，以及预训练 → 全量 SFT → DPO 偏好对齐全套流程，并配套 DeepSpeed/DDP 与 SwanLab 等监控。

目标：在可控算力下，完成「数据 → tokenizer → 清洗 → 预训练 → 指令微调 → 偏好对齐」的可复现流水线，而不是单点脚本堆砌。
定位：偏 工程化训练平台 + 小型模型实现，适合强调 数据工程、训练框架集成、模型侧关键模块。
技术栈： PyTorch · Transformers · TRL（DPO）· Accelerate · DeepSpeed· Datasets · pytest · SwanLab · 预处理 fasttext / bertopic 等。

模型侧：小模型但结构完整

因果 Decoder 架构：Embedding → N 层 Decoder → RMSNorm，对接 Transformers PreTrainedModel 生态。
MoE：可配置 routed/shared experts、top-k、辅助损失（aux_loss）等，训练时与 CE 联合优化。
RoPE + GQA（KV 头压缩）、Flash Attention、推理 KV Cache，与长上下文配置（如 max_position_embeddings）一致。
数据与分词：从语料到可训练格式

Tokenizer：从预训练 text 与 SFT conversations 汇总语料 → 训练 BBPE → 导出 Hugging Face FastTokenizer（含 chat template 与特殊 token 对齐）。
预处理：统一 CLI，预训练侧重质量过滤 → 精确/近似去重 → 可选 PPL；SFT 侧重对话结构、tool_calls 修复、拒答与质量过滤、去重与诊断
数据集与 Collator：高效训练细节

PreTrainDataset：文档流式拼接、<|endoftext|> 分隔、按 bin 做 packing。
SFTDataset：经 chat template 编码，labels 只监督 assistant，并做 first-fit packing。
DPODataset：兼容对话偏好对与扁平三元组，统一成 DPO 所需字段。
DataCollator：动态 padding；在 packing 分隔符处重置 position_ids；attention 拆成 prefix / causal block（prefix 不参与 loss 且可被监督段看见）。
训练阶段：工业界常见三阶段

CPT 预训练 → 全量 SFT → DPO；脚本层 STAGE=pretrain|sft|dpo + 单卡 / DDP / DeepSpeed。
监控：loss 归一化、梯度裁剪、熵与 top-1、固定 prompt 生成探针；预训练侧 MoE/隐藏层诊断，SFT 侧分项 loss、工具 JSON 可解析率等
